# systemhack_nov2025
Командное решение для задачи от МТС Финтех для хакатона "Система Хак"

## Текст задания

Сегодня многие компании малого и среднего бизнеса создают свои сайты, но часто они оказываются неудобными или даже полностью недоступными для людей с особыми потребностями. Это ограничивает аудиторию, снижает доверие и лишает пользователей возможности взаимодействовать с продуктами и сервисами.

Ваша задача – разработать инструмент, который поможет владельцам сайтов быстро и просто проверить, насколько их сайт соответствует базовым принципам доступности.

### Основные задания в рамках хакатона:

* Реализовать проверку корректности структуры и тегирования сайта, чтобы скринридеры (VoiceOver для macOS/iOS, NVDA или JAWS) могли корректно воспринимать и озвучивать контент.
* Сформировать понятные рекомендации по улучшению тегирования и структуры (например, добавление aria-атрибутов, корректное использование заголовков, описание изображений).
* Инструмент должен работать по простому сценарию: пользователь вводит URL, получает анализ сайта, отчет с результатами проверки и рекомендации по улучшению.

### Отчет может включать:

* Краткую оценку общего уровня доступности;
* Список найденных проблем с указанием конкретных элементов страницы;
* Пояснения, почему эти элементы создают трудности при работе со скринридерами;
* Рекомендации по исправлению

### Направления, которые необходимо реализовать:

* Автоматическая проверка: анализ структуры страницы, правильности использования тегов
* Формирование отчета: визуализация проблемных мест и рекомендации по их исправлению простым языком.
* Подсказки и примеры: что именно нужно изменить, чтобы сайт стал удобнее.
* Формат реализации:

**Вы сами выбираете формат реализации:**

* Веб-сервис: сайт, где пользователь вводит URL и получает отчет.
* Браузерное расширение: плагин, который сразу показывает анализ при открытии страницы.
* Гибридное решение – например, расширение с выгрузкой отчета на сайт.

**Главное условие: результат должен быть понятен и полезен не только разработчикам, но и обычным владельцам сайтов без технического бэкграунда.**

### Ограничения:

* Не нужно реализовывать полноценный поисковик или индексировать весь сайт. Работайте только с введенным пользователем URL.
* Ваша цель: дать пользователю максимально понятный и прикладной результат анализа, а не перегрузить его сложными техническими данными.
* При необходимости вы можете использовать готовые AI решения

### Дополнительно:

Если вы успешно сделаете базовый функционал, вы можете расширить решение:

* Проверка контрастности текста и фона.
* Анализ наличия и корректности альтернативного текста изображений.
* Подсказки по структуре страницы (заголовки, списки, формы).
* Автоматическая генерация рекомендаций с использованием готовых AI-моделей (например, автогенерация alt-текста).

## Инструкция к запуску

1. Установите Python 3.11.5
2. Введите в командной строке
   `pip install flask requests langchain-gigachat`
3. Создайте файл окружения `.env` в той же директории, где находится проект, и добавьте ключ доступа от [Gigachat API](https://developers.sber.ru/portal/products/gigachat-api)</br>
   Должно выглядеть так `SBERBANK_API_KEY="..."`
4. Запустите `app.py` и перейдите в браузере по ссылке `http://127.0.0.1:5000/`
